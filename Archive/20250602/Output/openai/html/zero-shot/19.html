In uploading files from a Mac to a Windows system, certain characters that are not valid in Windows file names are "escaped". For example a forward slash (/) is converted to '%2F'. Other such characters go unescaped and cause transfers to fail. Examples are asterisks (*) and double quotes (").

Since Fetch is capable of escaping some illegal characters, is there any way to get it to do the others?

Also, is there some way to provide a more desireable substitution. For example, I would prefer to have the forward slashes wind up as dashes.
